The Visual Display of Quantitative Information

Summary

The first part of this book reviews the graphical practice of the two centuries, and the second part provides a language for discussing graphics and a practical theory of data graphics. It’s concerned both with design and with statistics, but it is also about how to communicate information through words, numbers, and pictures.

[Graph integrity]
The visual representation of the data must be consistent with the numerical representation.
Lie Factor = size of effect shown in graphic / size of effect in data

An extreme example:

The magnitude of the change from 1978 to 1985 as shown in the graph is 783%, while the numerical change is 53%. Therefore the Lie Factor = 783/53 = 14.8, which is too big.

Firstly, a graph should display data variaion, not design variation. For example, a chaning of unit at the end is a design vatiation. In time-series displays of money, deflated and standardlized units of monetary measurementt are nearly always better than nominal units.

Secondly, using two varying dimensions to show one-dimensional data is a week and inefficient technique.
The number of variable dimensions depicted should not exceed the number of dimensions in the data.

If the area of the dollar is accureately reflected, then the 1978 dollar should be about twice as big as that shown.

Thirdly, graphics need to preserve the context of data. Different range of data chosen can result in completely different intepretations.

Part 2: Theory of Data Graphics

Data-Ink
A large share of ink should present data information.
Data-ink ratio = data-ink / total ink used to print the graphc.

An example of a graph whose ink is devoted mostly to non-data info:

As a result, the grid sea overwhelms the numbers.

In order to maximize the data-ink ratio, there are two erasing principles:

  1. Erase non-data-ink within reason

  2. Remove redundant data-ink within reason
    For symmetrical data such as box plot and open bar, it’s enough to present the left half.

    However, some times it’s necessary to create redundant information for convenience. For examle, for the train schedule graph above, it’s better to attach an extra half cycle to such that it’s easy to trace the time.
